sudo:

. . .is a program for Unix-like computer operating systems that allows users to run programs with the security privileges of another user, by default the superuser. -Wikipedia

If your user has administrative permissions you can use sudo to escalate your privileges to run commands you otherwise wouldn't be allowed to run.


XKCD #149 © Randall Munroe

commands:

  • sudo COMMAND
    • Attempt to escalate the privileges when executing COMMMAND.
  • sudo -i
    • Run the default shell as the root user.

  • su - USERNAME
    • Get a login shell for another user.
    • You will need the users password.

For the rest of the commands to run them as root use the sudo command.

users and groups:

Commands to get more information of and to manage users and groups.

commands:

  • who
    • list all of the users logged into the system
  • who am i
    • list which user you are

  • id
    • print user ID; group memberships and IDs

commands: continued

  • addgroup GROUPNAME
    • Create a new group.
  • deluser GROUPNAME
    • Delete an existing group.

  • adduser USERNAME
    • Create a new user.
  • deluser USERNAME
    • Delete an existing user.

  • usermod -aG GROUPNAME USERNAME
    • Add a user to a group.

files:

  • cat /etc/passwd
    • Where the users are stored
  • sudo cat /etc/shadow
    • The file where users encrypted paswords are stored

  • /etc/group
    • Groups and group memebership

network: interfaces, routes, nameservers:

Common commands used in basic network troubleshooting.

commands - interfaces:

  • ifconfig -a
    • List all of the network interfaces
  • ifconfig INTERFACE
    • List a single network interface

  • ip addr
    • Another way to list the interfaces

  • nmcli dev list iface INTERFACE | grep IP4.ADDRESS
    • A third way to get interface information

commands - routes:

  • netstat -rn
    • List route information

  • ip route
    • Also list route information

commands - nameservers:

  • cat /etc/resolv.conf
    • List nameservers

  • nmcli dev list iface INTERFACE | grep IP4.D
    • Also list nameservers

processes:

Listing and communicating with processes.

commands:

  • ps
    • A short list of tasks being run by the current shell including PID
  • ps -f
    • An expanded view including timing and the PPID
  • ps -fu root
  • ps -fu 1001
    • List all process for a user
  • ps auxwww
    • List all process with the expanded command-line

commands: continued

  • kill PID
  • kill -15 PID
    • Send the process a SIGTERM asking the process to quit.
    • Some programs will try to exit gracefully with unlikely data loss
  • kill -3 PID
    • Send the process a SIGQUIT asking the process to quit and perform a core dump
  • kill -9
    • Send the process a SIGKILL forcing the application to immediate exit
    • Likely data loss.

commands: continued

  • pgrep COMMAND
    • List matching COMMANDs that are running
  • pgrep COMMAND
    • List matching COMMANDs that are running and list the command name
  • pkill COMMAND
    • Search for and send COMMANDs a SIGTERM
  • pkill -9 COMMAND
    • Search for and send COMMANDs a SIGKILL

archives:

For backups or just saving space.

commands:

  • tar cvf foo.tar PATH/FILES
    • archive files or a path
  • tar cvfz foo.gz PATH/FILES
    • archive and compress files or a path
  • tar xvfz foo.tar
    • unarchive files and directories from an archive
  • tar xvfz foo.gz
    • unarchive and decompress files and directories from an archive

commands: continued

  • zip -r foo PATH/FILES
    • Compress files/directories
  • unzip foo.zip
    • Uncompress files/directories
