sudo and su

We covered the root user last week. sudo is important so we will cover some more. Right now we want to focus that we can use sudo to execute commands or to switch users to execute multiple commands as the root user. 


sudo top


sudo -i # use sudo to start an interactive shell

sudo su - # use sudo to switch the user

su - # switch user with the root users password

IO Streams in Unix applications

Many of the applicaitons that we use in a Linux/Unix CLI will take in and process data via STDIN and will emit data as STDOUT and STDERR. All three of these can be thought of as streams of Input/Ouput data.

  • STDIN
    • Standard In
  • STDOUT
    • Standard Output
  • STDERR
    • Standard Error

pipes

Pipes allow you to pass data between applications to do more advanced processing. 


# the output of ls -al will be sent as input to more which allows you to view one page at a time.
ls -al | more

# take the output of ls -al and only display files that contain the string "Jan"
ls -al | grep Jan

redirection

Redirection allows for you either feed data from a file into an application or to take data from an application and write it to a file. 


# create or overwrite a file with the output from the ls command.
ls -al > mylist

# create or append to a file with the output from the ls command.
ls -al >> myList

# sort contents of file list.txt
sort < list.txt

grep

grep allows you to print lines from a file or a datastream that match a string. 


# find all the lines in passwd file that include your username
grep yourusername /etc/passwd

# find the root user in /etc/passwd and /etc/group
grep root /etc/passwd /etc/group

# find the users in /etc/passwd that are not root.
grep -v root /etc/passwd

# take the output from 'cat /etc/passwd' and match lines with 'root'.
cat /etc/passwd | grep root

sort

sort allows for you to organize data. 


# sort the output of 'ls -tr' command in alphabetical order.
ls -tr | sort

# sort the output of 'ls -tr' command in reverse alphabetical order.
ls -tr | sort -r

# Sort the long output of the ls command based on the filename(nineth key).
ls -latr | sort -k 9

find

Find files based on your criteria. 


# find all files and directories under / named passwd
find / -name passwd

# find all files and directories under /home owned by the user linuxadmin
find /home -user linuxadmin

# find all directories in / owned by the user linuxadmin
find / -user linuxadmin -type d

# find all files in / that belong to the root user and are named list.txt.
find / -user root -type f -name list.txt
