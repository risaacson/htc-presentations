Kernel Overview

  • The Kernel is the main component of most computer operating systems.
  • It is a bridge between applications and the actuall data processing done at the hardware level.
  • The kernel's responsabilities include managing system resources.

Linux Kernel History

  • April 1991 Linus Torvalds initially conceived and created what would become the Linux Kernel.
  • Augest 25th 1991 Linus made a public announcement that he was working on a free operating system.
  • From 1991 to now over 12,000 programmers have contributed to the source.
  • September 1991 Kernel version 0.01 was realeased.
  • March 26th 2016 Kernel version 4.6-RC1

Linux Kernel Info


uname -o # Prints the Operating System
uname -r # Prints the Kernel Release
uname -v # Prints the Kernel Version
uname -a # Prints all Kernel Information.

Kernel Modules

  • Kernel Modules are able to be added or removed to ad or remove functionality without usually requireing the Operating System to be rebooted.
  • Located under /lib/modules/KERNELVERSION/ as *.ko files.

Kernel Modules & Kernel Info


# Run all commands as root or use `sudo`

lshw # Print out the existing hardware.
lspci -k # List PCI devices and drivers.

apt-get install kvm # Install KVM
lsmod | grep kvm # Search if KVM is installed in the kernel. It should not be.
modprobe kvm # Load the KVM module into the kernel.
lsmod | grep kvm # Search if KVM is installed in the kernel. It should be.
rmmod kvm # Remove the KVM module from the kernel.
lsmod | grep kvm # Search if KVM is installed in the kernel. It should not be.
apt-get purge kvm && apt-get autoremove -y # Clean-up after yourself!

Upgrading the Kernel

  • Ubuntu/Redhat/OpenSUSE
    • Update your installed packages and the package manager will take care of upgrading the kernel and modules for you. (Deps)
  • From Source(Manual)
    • Download and Uncompress the Kernel Release (source code)
    • Run the configure step to define what features you need
    • Compile and install the kernel and modules.
    • Update the bootloader to point to the new kernel.

Logging

  • Logging alows you to get output from system servies or daemons that do not run in the foreground
  • Older Linux distributions used syslog while modern Linux distributions use rsyslog
  • Rsyslog runs as rsyslogd and is configured in /etc/rsyslog.conf with logging rules in /etc/rsyslog.d
  • After you change the configuration you can restart the service with 'sudo restart service'
  • By default syslog and rsyslog write out under /var/log/
  • Log rotation is configured out of the box so your disks don't fill up.

Common Log Files


# To read the logfiles you may need super-user permissions.

/var/log/syslog # Logs everything except authorization related messages
/var/log/auth # Logs Auth related messages
/var/log/boot # Logs boot related messages
/var/log/dmesg # logs kernel and driver related messages
/var/log/mail # logs mail related messages

# Using 'man FILENAME' will typically yield information about what
# the logfile is for.

Logger

  • The 'logger' command allows you to write out log message using the system facilities
    • 
logger Admin Making a Change # The message is logged out
                             # into /var/log/syslog
  • Use 'man logger' for information on the various options.
