Shell Scripting Basics

Create a file named hello.sh. 


#!/bin/bash
echo "Hello, World"

Set the Execute Bit 


chmod +x hello.sh

Execute the script. 


./hello.sh

Shell Scripting Basics Continued...File


#!/bin/bash

If the execute bit it set on a file and you try to run it bash will look at the first file to figure what to execute it with. Above we have told bash to execute our script with bash. 


echo "Hello, World"

The echo command takes a string and writes it to output.

Shell Scripting Basics Continued...File

  • What is a script?
    • text file
    • Starts with a "#!" line
    • Has the execute bit set
    • Follow the syntax of the scripting language you have chosen
  • Why use a script?
    • If you run it twice you should automate it.
    • It allows you to save and easialy recall logic.

    Avoid naming the same as commands built on the sytem. Also commands work the same as they work on the command line. Case matters.

Variables

Variables allow you to store a value for later usage. 


#!/bin/bash

MESSAGE='Hello, World'
aDATE=$( date "+%Y%m%d%H%M" )

echo ${MESSAGE}
echo "A Date: ${aDate}"
  • Quote Strings. Single if you have no variable replacement. Double if you have variable in the string.
  • Don't have a spaces between the variable the equals and the value you are setting.
  • Set variables from a sub-command by either using backticks or $( COMMAND )

Math Operations


#!/bin/bash

x=5
y=10

sum=$(( x + y ))

echo "${x} + ${y} = ${sum}"

Execute a Math Operation inside $(( )).

  • + = plus, - = minus
  • * = multiply, / = divide
  • ** = exponent, % = modulo/remainder

if Statements: conditionals/tests

If statements provide scripts with the ability to have logic. If this, then do that. 


#!/bin/bash

echo "Type \"yes\" or \"no\""
read yesno

if [ "${yesno}" -eq "yes" ]; then
  echo "You chose yes."
else if [ "${yesno}" -eq "no" ]; then
  echo "You chose no."
else
  echo "Please \"yes\" or \"no\""
fi
