Anti-Virus

For the most part the way that Linux has been engineered makes it harder for viruses to target it. In the future Linux is projected to become a bigger target. In the meantime running an anti-virus applicaion helps protect other systems.

Ultimately making sure that your system has up to date software is some of the best protection. Scanning with anti-virus software is a bonus along with scanning for rootkits.

Anti-Virus examples. . .

  • ClamAV
    • Open Source
    • ClamTk

  • Comodo Anti-Virus for Linux
  • Avast! Linux Home Edition
  • AVG Antivirus

Disk Encryption

Full disk encryption protects your data by cryptographically encoding your data requiring a passphrase to unlock it. Though if you lose your passphrase your data is very difficult or impossible to reasonably recover.

Use this to prevent from unauthorized access to your data. For example booting from an alternate disk and mounting your devices and accessing your data.

Firewalls

The Linux firewall lives as a part of the kernel and is a packet filter. In the kernel it is netfilter, from the cli you traditionally use iptables, and now you can use ufw(uncomplicated firewall) for easier use. If you want a GUI interface you can use GUFW.

Practically what is happening is that netfilter inspects packets coming in to the network interfaces and if a packet matches a rule you can allow, delete, or reject.

This is a big deal for security because if you have a firewall in place you can block traffic from an attacker before it talks applications on your computer.

Demos


sudo apt-get install ufw

# Get the status
sudo ufw status

# Enable/Disable
sudo ufw disable
sudo ufw enable

# Set Up Defaults
sudo ufw default deny incoming
sudo ufw default allow outgoing

# Allow http
sudo ufw allow http

# Reset rules
sudo ufw reset

Passwords

Your password is your first line of security so choose wisely.

When choosing a password first try to understand the password requirements and limitations.

If you need to use a short password use both upper and lower case letters and numbers. If possible use special characters and whitespaces.

If you are able to use a longer password try to choose a passphrase with words separated by whitespaces which may be easier to remember.

The Krebs Dos and Don'ts is good to keep in mind. http://krebsonsecurity.com/password-dos-and-donts/

https://howsecureismypassword.net/ can give you an idea of how hard it would be to crack a password. Don't use your password here, just examples.

File Encryption

We are going to look at encrypting files so that we can safely transfer them to people that we trust with the information.

Glossery

  • Encryption
    • The process of encrypting data in such a was that unintended recipients cannot read it.
  • Plaintext
    • The original or decrypted message.
  • Ciphertext
    • The encrypted message.
  • Algorithm
    • A set of instructions that are performed to encrypt and decrypt a message.

Glossery - continued.

  • Key
    • A value which is used by algorithm to perform encryption of a certain key.
  • Key Ring
    • Hold your keys and other users' public keys.
  • Symmetric Encryption
    • Uses the same key to encode and decode information.
  • Asymmetric Encryption
    • Uses different keys to encode and decode the information.
    • Public Key Encryption is an example.

GNU Privacy Guard(GPG)

  • A free and open source software used to generate keys as well as to encrypt and decrypt data.
  • Built into some Linux distributions or installable through the package manager.
  • Find more info using `man gpg` or at http://www.gnupg.org/gph/en/manual.html

Symmetric Encryption


# Create a file
echo "foo bar baz" > aFile.txt

# Encrypt the file and you will be asked for a passphrase.
gpg -c aFile.txt

# list and get the type of the file
ls aFile.txt*
file aFile.txt.gpg

# move the original file, decrypt and compare
mv aFile.txt.orig
gpg aFile.txt.gpg -o aFile.txt.new
diff aFile.txt.orig aFile.txt.new

Asymmetric Encryption


# Create Alice as a User
sudo useradd alice
sudo su - alice
# Genertate a Key for Alice
# Enter in Kind of Key (RSA and RSA), Size (2048), Expires (0)
# Real Name (Alice), E-mail (alice@example.com), Passphrase (alice)
gpg --gen-key
# Export Alice's public key.
gpg --armor --output /tmp/key-pub-alice.asc --export alice
exit

Asymmetric Encryption - Continued


# Create Bob as a User
sudo useradd bob
sudo su - bob
# Genertate a Key for Bob
gpg --gen-key
# Export Bob's public key.
gpg --armor --output /tmp/key-pub-bob.asc --export bob
exit

Asymmetric Encryption - Continued


# As Alice import Bob's Key
sudo su - alice
gpg --import /tmp/key-pub-bob.asc
exit

# As Bob import Alice's Key
sudo su - bob
gpg --import /tmp/key-pub-alice.asc
exit

Asymmetric Encryption - Continued


# List Keys as Alice
sudo useradd alice

# List Public Keys
gpg --list-keys

# List Secret Keys
gpg --list-secret-keys
exit

Asymmetric Encryption - Continued


# Have Alice create a file to encrypt
sudo su - alice
# Run this a couple times
echo "foo bar baz" >> aFile.txt

# Encrypt the file to bob
gpg --encrypt aFile.txt
# Type in bob as the ID, confirm with a y, and hit enter to exit.

cp aFile.txt.gpg /tmp/aFile.txt.gpg
exit

Asymmetric Encryption - Continued


# Decrypt a file as Bob.
sudo su - bob

gpg --decrypt -o ~/aFile.txt /tmp/aFile.txt.gpg
# Enter bob's passphrase

cat aFile.txt

exit

Asymmetric Encryption - Continued


# Delete Keys:
sudo su - alice

# Delete a public Key
gpg --delete-keys bob

# Delete a Secret Key
gpg --delete-secret-keys alice

exit

Asymmetric Encryption - Continued


# Clean-up:
sudo  deluser alice --remove-all-files
sudo  deluser bob --remove-all-files
sudo  sudo rm /tmp/aFile* /tmp/key-pub-*
