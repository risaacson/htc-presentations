Intro to the CLI

CLI

Command-Line Interface

A text driven interface where you type (or cut-and-paste) in commands to be executed.

Tonight we will focus on:

  • Accessing the CLI
  • The CLI and the shell
  • Basic shell commands
  • Basic editor usage

We are not going to go too in depth as the Linux Admin classes should dive in deeper.

The CLI and the Shell

When you open up a terminal program it opens up your default shell. It it through a shell that you are able to enter in your commands.

You can view your availible shells via `cat /etc/shells`.

Your default shell is set with your user information in /etc/passwd

For now just be concerned that shells allow you to:

  • execute commands against your system
  • compose commands
  • script logic
  • set re-usable environmental variables

The shell you choose is a religious decision. When you make a choice it becomes a part of your identity.

Which shell you prefer doesn't matter as they all provide the same basic features and the differences are syntax.


In computing the same can be said for editors, programming languages, operating systems, amongst other things.

There are many shells that we can use. The most popular and typical default shell is BASH. We will be using BASH.

Other examples of shells are:

  • Bourne Shell - sh
  • ksh
  • csh
  • zsh
  • fish

Performs similar functions to cmd.exe and PowerShell.

BASH

BASH is the Bourne-again Shell which is a free rewrite of the Bourne Shell with additional features. The software is hosted by the GNU Foundation and it is available on just about every Unix or Linux distribution.

It has command name, path, command argument(if configured), and wildcard completions. Also it will keep an easy to access command history. Also it has a great set of structures to make scripting powerful.

Basic Shell Commands

  • If you have questions about a command:
    • man COMMAND
    • info COMMAND
    • or google the command
  • Your command history:
    • history -- print your history
    • history | grep STRING -- search your history

A Filesystem Aside

The Linux and Unix filesystems are interacted through a heierarcy. You have a root (/) directory and everything is nested in folders.

The path is how you get from the root to something. You can also have a relative path.

The PATH environment variable helps execute commands easily.

Commands:

  • pwd -- print your current directory
  • echo "${PATH}" -- print the path environment variable.

Basic Shell Commands Cont. . :

  • Listing
    • ls -- list the contents of the current directory
    • ls PATH/FILE -- list the contents of the directory or file
    • ls -l -- list in long format
    • ls -a -- list all files (including hidden)
    • ls -t -- list sorted order by modification time
    • ls -r -- list in reversed order
    • ls -latr -- list all files by what was recently modified

Basic Shell Commands Cont. . :

  • Moving Around
    • cd -- change (working) directory, by default to your home
    • cd / -- change to the root directory
    • cd ${HOME} -- change to your home directory variable replacement
  • Creating Directories
    • mkdir foo -- create a directory
    • mkdir -p path/foo -- create a directory and any directorys that don't exist in the path
  • Creating a File
    • touch bar -- create an empty file

Basic Shell Commands Cont. . :

  • Deleting Files and Directories
    • rmdir foo -- remove an empty directory
    • rm bar -- remove a file
    • rm -r foo -- recursively remove directories and files

Basic Shell Commands Cont. . :

  • Copy and Move Files
    • cp foo /tmp/ -- copy foo to the /tmp directory
    • cp foo foo2 -- copy foo to foo2
    • mv foo /tmp/ -- move foo to the /tmp directory
    • mv foo foo2 -- rename the file foo to foo2

Basic Editor Usage

Editors(a limited list):

vi

Vi has two modes insertion mode and command mode. When you enter the editor you are in command mode. You use insertion mode to make changes. After you are done with insertion mode you use [ESC] to return to command mode.

Find a cheat sheet that you like, print it out, and keep it next to your computer until you use the commands enough to remember them.

Start by typing 'vi FILENAME'

vi - basic commands

  • Quitting and Saving
    • :q -- exit if no changes have been made
    • :wq or :x -- exit saving(writing) changes
    • :q! -- exit not saving changes
  • Moving Around
    • up arrow or 'k' -- move up
    • down arrow or 'j' -- move down
    • left arrow or 'h' -- move left
    • right arrow or 'l' -- move right

vi - basic commands Continuted. . .

  • Inserting Text
    • i -- insert before cursor
    • a -- append after cursor
    • o -- append in a new line after the current line
    • r -- replace a character
    • cw -- change word
  • Deleting Text
    • x -- delete a character to the right of the cursor
    • dd -- delete the current line

vi - basic commands Continued. . .

  • Yanking and Putting Text
    • yy -- yank the current line (copy)
    • dd -- delete the current line (cut)
    • p -- append after the position or after the line
    • P -- insert before the position or before the line
  • Searching
    • /STRING -- search for the first instance after cursor of a STRING (wraps)
    • n -- search again for the next instance
    • N -- search again for the previous instance

nano

Super basic editor that is appropriate for beginners. All of the commands are based off of using control keys.

The commands try to help you out and clarify what you are trying to do.

Start by typing 'nano FILENAME'

nano - basic commands

  • CTL-x -- exit
  • CTL-o -- save your file
  • CTL-w -- search for text

GUI Based Editor

Your distribution along with installing a GUI will install editors that you can use to create and modify files. You can start these application by using commands from a shell.

The easiest method to use if you have a GUI.

  • Ubuntu
    • gedit FILENAME
  • Kubuntu
    • kate FILENAME
  • Lubuntu
    • leafpad FILENAME

Often I use Atom from the fine folks at GitHub.
