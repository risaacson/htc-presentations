    The Secure Shell Protocol (SSH) is a protocol for secure remote login and other secure network services over an insecure network. -RFC4252

    History

    • 1995, Tatu Yl√∂nen, Helsinki University of Technology researcher, implemented SSH-1
    • 1999, developers wanted free software ended up creating OpenSSH from the OpenBSD community
    • 2006, A revised version of SSH-2 became a standard.

    • References:
      • Secure Shell (wikipedia)
      • OpenSSH open source implementation

    SSH

    • Allows for you to connect and be given a remote shell.

    • OpenSSH is commonly used. Putty is a Windows client.
    
ssh -V

ssh user@host.tld

ssh -i ~/.ssh/id_rsa user@host.tld

ssh host.tld "hostname"

man ssh

    SSH-KEYGEN

    
ssh-keygen -t rsa -b 4096 -C "user@host.tld"

    SSH-ADD

    • You can add keys to your keyring.

    • We saw this when we were being introducted to Ansible.
    
ssh-add key.pem

man ssh-add

    SSH-COPY-ID

    • Used to copy your ssh public key to a remote host and set permissions.
    
ssh-copy-id user@host.tld

ssh-copy-id -i ~/.ssh/id_rsa user@host.tld

cat ~/.ssh/id_rsa.pub | ssh user@host.tld "mkdir -p ~/.ssh && cat >>  ~/.ssh/authorized_keys"

man ssh-copy-id

    SCP

    • Secure copy of files and directories.
    
scp aFile user@host.tld:aDirectory/aFile

scp user@host.tld:aDirectory/aFile aFile

scp -r user@host.tld:aDirectory ./

man scp

    RSYNC

    • Sync files & permissions between locations.
    • For remote files works over SSH.
    
rsync -a aFile user@host.tld:aDirectory/aFile

man rsync
